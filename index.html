🐙 Testing-Library

Esta enfocado en probar lo que el navegador renderiza, el resultado de JavaScript.

  • No, no es un test runner, como Jest
  • No, no es un Framework de tests
  • Es una librería de utilidades

Recomendaciones de testing

De personas que de verdad saben de esto

  • Los test son agnosticos a cambios en el código
  • Se prueba el funcionamiento esperado
  • Un código más testeado es un código más confiable
  • La descripción del test es la acción esperada
  • La descripción puede contener o no algún identificador
“Un código testeado es un código que tiene menos posibilidades de fallar en producción, porque ya lo probaste en tu nave espacial.”
